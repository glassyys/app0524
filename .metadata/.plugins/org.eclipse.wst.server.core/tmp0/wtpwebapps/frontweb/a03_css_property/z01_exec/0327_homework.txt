### 조별 토론 주제 (3문제)
1. **색상 코드 vs 색상 이름**: 
CSS에서 색상을 지정할 때 16진수 코드,
RGB 코드, 색상 이름 중 어떤 방식을 선호하는지 토론하세요.
각 방식의 장단점은 무엇인가요?
색상이름을 선호합니다.
1) 16진수 코드 
장점:
- 이해하기 쉽다
- 일반적으로 쓰이는 코드이며 경우의 수가 많다.
단점:
- 색상기억하기 어렵다
- 투명도를 지정할 수 없다
2) RGB 코드
장점: 
-다양한 색상 조합이 가능
-색상을 세부적으로 제어할 수 있고 투명도를 지정할 수 있다
단점:
- 코드를 보고 확인이 안됨
- 16진수보다 복잡할 수 있으며 코드 값을 이해하기 어려울 수 있다.
3) 색상 이름
장점: 
- 직관적인 이름을 가지고 있으며 코드의 가독성을 향상시킨다
단점: 
- 다양한 색상을 표현하기 어렵다.
- 경우의 수가 적으며 CSS 파일의 용량이 커질 수 있다.

2. **텍스트 꾸미기의 중요성**:
웹 페이지 디자인에서 `text-decoration`, `text-align`, `text-indent`와 
같은 텍스트 스타일링 속성이 왜 중요한지 토론하세요. 사용자 경험(UX)과 
접근성 측면에서의 영향을 중심으로 논의하세요.
답변:
웹 디자인에 따라 표현하고자 하는 내용을 쉽게 표현할 수 있고
색상, 정렬 상태, 폰트 등을 통해 시각성과 가독성을 향상시켜서
접근해야할 요소에 사용자가 쉽게 접근할 수 있다.

3. **단위 선택의 기준**: CSS에서 단위(px, em, % 등)를
선택할 때 어떤 기준으로 결정하는 것이 좋은지 토론하세요.
답변:
상황과 요구사항, 상대적 및 절대적 크기, 반응형 디자인, 가독성 및 접근성이 좋다
각 단위별로 보면 아래와 같습니다.
1) px은 절대 단위이므로 상황에 따라 역동적 특성이 없다
-> 코드가 고정적으로 유지되어야 할 때 사용하는 것이 좋다
2) %는 유동적이고 부모 요소의 크기에 상대적으로 움직인다.
비율의 크기를 쉽게 비교할 수 있다.
3) em은 상대적인 길이 단위로 폰트 크기에 영향을 받는다. 
특히 em은 부모 요소의 폰트 크기를 
상속받기에 부모 요소가 바뀔 때 함께 바뀌어야하는 상황에서 사용하면 좋다.

반응형 웹 디자인에서 단위 선택이 왜 중요한가요?
답변:
1) 알맞은 단위 선택을 통해 반응형 웹 디자인에서 유동적인 디자인을 가질 수 있고,
성능 최적화를 할 수 있으며 가독성과 접근성을 향상시키면서
사용자 친화적인 디자인을 구현할 수 있다.
2) 브라우저 환경이 다양한데 환경에 맞게 구현이 가능합니다.
모바일의 경우 특히 장점이 됩니다.

1) 16진수 코드 (Hexadecimal Color Code):
장점:
간결하고 이해하기 쉽습니다.
웹에서 가장 일반적으로 사용되는 방법 중 하나입니다.
다양한 색상을 정확하게 표현할 수 있습니다.
단점:
일반적으로 사람이 외우기에는 복잡합니다.
특히 특정 색상을 기억하는 데는 어려울 수 있습니다.
2) RGB 코드 (RGB Color Code):
장점:
색상을 빨강, 초록, 파랑의 세 가지 채널로 나누어 표현하므로 직관적입니다.
다양한 색상을 조합하여 만들 수 있습니다.
프로그래밍적으로 생성 및 조작하기 쉽습니다.
단점:
가독성이 16진수보다 떨어질 수 있습니다.
투명도를 나타내는 알파 채널이 없습니다.
3) 색상 이름 (Color Name):
장점:
기억하기 쉽고, 읽기 쉽습니다.
가독성이 뛰어나며 적용이 쉽습니다.
단점:
색상의 이름을 정확하게 기억해야 합니다.
미리 정의된 색상 목록에만 제한됩니다.
특정 색상을 정확하게 표현하지 못할 수 있습니다.
일반적으로는 웹 개발에서는 16진수 코드나 RGB 코드가 가장 일반적으로 사용됩니다. 
이는 웹 페이지에서 색상을 정확하게 제어할 수 있고, 특히 디자인 작업에서 색상을 고려할 때 유용합니다. 하지만 색상 이름은 가독성이 좋고 기억하기 쉬우므로 간단한 경우나 코드의 가독성을 중요하게 생각할 때 유용할 수 있습니다.
