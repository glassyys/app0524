2. `z-index`가 작동하지 않는 경우, 어떤 조건이 충족되어야 하는지 논의하세요.
1) 작동하지 않는 경우
- Position 속성이 설정되지 않은 경우
- 위치 지정 속성이 static인 경우
- 지정값의 순서가 잘못된 경우
- 부모요소의 값의 관계에 따라 z-index의 우선순위가 변경될수 있다.
2) 어떤 조건이 충족되어야 하는가
- 속성은 위치 지정 속성인 position 
(예: relative, absolute, fixed, sticky)이 설정된 요소로만 적용 필요
- 지정값의 순서를 명확하게 하는 것 필요
- 부모요소의 값에 영향을 받으므로 비교되는 범위를 잘 알고 사용해야 한다.

3. `visibility: hidden;`과 `display: none;` 사이의 주요 차이점은 무엇이며,
어떤 상황에서 각각을 선호하나요?
1) "visibility: hidden" 의 경우 값이 차지하는 공간이 유지되지만
"display: none;" 경우 값이 차지하는 공간이 삭제된 것처럼 사라진다.
2) "visibility: hidden" 의 경우 값이 마우스 이벤트 같은 이벤트에 반응하지만
"display: none;" 반응하지 않는다.

상황에 따라 선호하는 것은 달라질수 있습니다.
보안적인 이슈가 있을때는 "display: none;" 이 유리하고
문제에 대한 정답을 제공해야 하는 경우는 
"visibility: hidden"을 사용해야 유리하다.

4. `overflow: auto;`와 `overflow: scroll;`의 차이점을 설명하고,
각각을 선호하는 상황은 어떤 것이 있나요?
1) overflow: auto : 내용이 크기를 초과하면 스크롤바가 표시되고 그렇지 않으면 표시안 됨
2) overflow: scroll; 내용의 크기와 상관없이 스크롤바가 반드시 표시됨

"overflow: auto :"를 선호하는 경우는 레이아웃의 안정성을 유지하고자 할때나
내용이 넘치는 경우가 적은 경우
"overflow: scroll;"를 사용자가 스크롤을 할수 있게 하여 
페이지 사용 경험을 향상시킬 경우 선호합니다.
