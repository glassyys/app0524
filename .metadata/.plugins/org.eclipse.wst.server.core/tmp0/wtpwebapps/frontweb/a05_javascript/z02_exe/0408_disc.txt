2. **순차구조의 중요성**
   - 순차구조는 프로그램의 기본 흐름을 정의합니다.
실제 프로젝트에서 순차구조를 효과적으로 설계하는 방법과 그 중요성에 대해 논의해 보세요.
순차구조 설계 실패가 프로젝트에 미칠 수 있는 영향은 무엇인가요?
토론: 
1) 순차구조를 효과적으로 설계하는 방법
아래와 같은 방법이 있습니다.
- 욕심쟁이 기법 - 비교적 단순하면서도 널리 사용이 가능한 기법으로 선택해야 할 방법이 
여러 가지일 때 현재 상황에서 가장 최선인 것을 먼저 선택하는 방법.

- 분할 통치법 - 해결하려고 하는 문제를 작은 여러 개의 부분 문제로 나눈 뒤,
작은 부분의 문제부터 해답을 구한 다음 이를 통합하여 원래의 문제에 대한 해답을 얻는 방법.

2) 순차구조를 효과적으로 설계해야 하는 중요성
- 폭포수이론에 따라 순차구조는 수정하기 어렵기에 명확한 진행과정 설계없이 시작 했을때
  자원적 손실이 크게 발생할 수 있다. 
- 순차구조를 파악하기 위해서는 요구사항에 대한 분석이 잘 되어야 합니다.
- 고객이 요구하는 내용에 대해 명확히 이해하고 순차구조를 통해 설계해야지 고객이 원하는 결과가 나오고
요구사항 변경내용이 있을때 수정이 용이합니다.

3. **조건문의 선택 기준**
   - 다양한 조건문(`if`, `if else`, `if else if)을 사용하는 상황을 구분하여,
각 상황에서 가장 적합한 조건문 선택 기준에 대해 논의해 보세요.
어떤 경우에 특정 조건문을 사용하는 것이 더 효율적인가요?

3-1) if문
조건이 참일경우 단순하게 결과만 알려줄때인데 
if else와 if else if는 앞의 조건이 거짓일 때 실행되기에 여러 개의 조건이 참인 경우 실행되어야 하는 상황에서 사용할 수 있다.
조건이 참일 시 실행되는 코드 구현에 적합하다. 
예로는
사용자가 로그인 한 경우 로그인 성공결과만 확인할 때 사용 할 수 있습니다.

3-2)if else문
- 참인 조건이 모두 만족하지 않은 경우에 false 출력값이 필요할 때 적합하다.
참인 조건이 여러개인 경우 다 출력하고 싶을때 사용합니다.
- 여러가지 경우 중 선택하여 결과를 리턴할때입니다.
놀이공원 할인이냐 아니냐? 또는 출입 가능여부 판단할때


3-3)if else if의 경우는 성적에 대한 등급이 여러가지 결과가 나오는 경우입니다.
-이때 앞선 조건이 flase일 때 다음 조건이 실행되므로 
고려하는 조건이 겹치지 않아야하는 상황에서 적합하다.
예로는 성적에 대한 등급을 표시할때 입니다.
