### 조별 토론과제 

1. 자바스트립트에서 `break`와 `continue`의 차이점과 각각의 적절한 사용 시나리오를 논의하시오.
토론결과:
1-1) break를 반복문에서 즉시 처리를 종료함으로 특정조건이 충족되었을 때 종료하고자 할때 사용합니다.
continue 현재의 반복을 중지하고 다음 반복문을 처리합니다.
 
1-2) 적절한 사용 시나리오를
- break의 경우는 특정한 조건에 맞으면 처리를 종료할 경우
은행에서 비번 5회이상 틀릴시 강제 종료 및 계정 잠금
- continue 
물건을 구매할 때 사은품이나 서비스로 받아 바코드가 찍히지 않아야하는 경우
("무료"로 인식되는 경우를 제외하고 가격이 계산될 수 있도록)

2. `while`, `do-while`, `for` 각 반복문의 특징과 상황에 따른 최적의 사용 경우를 비교하시오.
토론결과:
while는 조건식이 만족할 경우 코드를 반복한다. 처음부터 조건을 만족하지 않으면 아예 실행이 되지 않는다
do-while 반드시 한번은 실행해야하는 경우 사용
예시)
설문조사시는 설문종료가 필요할경우 사용가능합니다
동의정보처리시 비동의시 처리를 중단할 수 있습니다.

for : 반복문을 실행하기 전에 초기화를 한 번 진행한 후 조건식을 만족하는지 여부에 따라 코드를 반복한다.
예시는 반복적인 계산이나 누적계산시 사용
카페와 같이 재고를 확인하면서 물건을 파는 곳에서 숫자를 카운트 해야할 때 for문 사용. 

3. 함수의 매개변수와 리턴값의 역할에 대해 토론하고, 효과적인 함수 디자인 예제를 제시하시오.
토론결과:
3-1) 함수의 매개변수와 리턴값의 역할
함수에 매개변수를 사용하면 함수에 입력값을 변경하여 호출하여 함수 로직에 맞는 결과를 받아 사용이 가능합니다.
리턴값을 통해 함수내부 로직에 의한 작업의 최종 결과값을 받을수 있게 된다.

3-2) 효과적인 함수 디자인 예제
- 함수의 이름과 설명이 명확해야 합니다.
- 매개변수에 대한 설명이 있어서 사용자가 어떤성격의 매개변수를 전달할지 이해가 쉬워야 합니다
- 간결한 코드를 사용하여 가독성이 쉬워 접근성이 좋아야 하고 유지보수가 쉬워야 합니다.






명확한 함수 목적: 함수의 이름과 설명이 명확하게 주어져 있어서 어떤 기능을 하는지 쉽게 이해할 수 있습니다.
매개변수 설명: 함수에는 매개변수에 대한 설명이 주어져 있어서 사용자가 어떤 종류의 인수를 전달해야 하는지 이해할 수 있습니다.
유효성 검사 로직 분리: 이메일 주소의 유효성을 검사하는 로직이 별도의 함수로 분리되어 있어서 함수의 재사용성이 높아집니다.
이 검사 로직은 다른 함수에서도 필요할 수 있으므로 분리하여 재사용할 수 있습니다.
간결한 코드: 함수는 한 가지 일을 수행하도록 설계되어 있어서 코드가 간결하고 가독성이 높습니다.
이러한 예제는 함수 디자인에서의 좋은 실천 방법을 보여줍니다. 함수의 목적을 명확히하고,
각 함수가 한 가지 일을 잘 수행하도록 설계함으로써 코드의 유지 보수성과 가독성을 향상시킬 수 있습니다.
